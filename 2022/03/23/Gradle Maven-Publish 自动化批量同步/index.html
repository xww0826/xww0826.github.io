Gradle Maven-Publish 自动化批量同步

本文最后更新于：3 个月前

前言

由于我们项目中有特殊需求，Module 数量超过30+。但是在做 Maven 版本维护时，就比较头疼。早期我们在每个 module 的 gradle 中引入的是 maven 插件，在同步到远程时由于 module 较多，所以每次都需要手动去 uploadarchives 略显繁琐！

所以，为了解决这个问题，我们将 maven 插件替换成了 maven-publish 插件，通过编写 gradle 代码与 shell 脚本实现批量自动化上传。

目录结构

project
|__ app
    |__ build.gradle
|__ lib_one                    
    |__ build.gradle
|__ lib_two                    
    |__ build.gradle                 
|__ build.gradle
|__ gradle.properties
|__ batchPublish.gradle
|__ batchPublish.sh

maven-publish 插件

一、首先是 batchPublish.gradle 中代码，这里是为了让每一个 module 引入 maven-publish 插件，填写你项目的远程 maven 仓库信息，用于上传目的：

plugins {
    id 'maven-publish'
}

afterEvaluate { project ->
    def groupId = project.ext["GROUP_ID"]
    def artifactId = project.ext["ARTIFACT_ID"]

    publishing {
        publications {
            release(MavenPublication) {
                from project.components.release
                it.groupId = groupId
                it.artifactId = artifactId
                it.version = '1.0.0'
            }
        }

        repositories {
            maven {
                allowInsecureProtocol true
                url = 'http:xxx'
                credentials {
                    username = 'xxx'
                    password = 'xxx'
                }
            }
        }
    }
}

二、然后在每个子 module 的 build.gradle 中引入：

apply from: '../batchPublish.gradle'

this.ext.GROUP_ID = "com.test.demo.plugin"
this.ext.ARTIFACT_ID = "test"

批量上传

新建 batchPublish.sh 文件，添加以下 shell 脚本，完成后双击即可上传（注意 Java 与 SDK 环境）

#!/bin/bash    
./gradlew publish    
exec /bin/bash
